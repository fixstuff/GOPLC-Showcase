Part VI covers the event bus and webhook delivery subsystem. Any significant state change inside a running GoPLC instance — protocol connection, fault, configuration update, program reload, scan overrun — is published to an internal event bus that can be subscribed to by ST code, streamed over WebSocket, dispatched to HTTP webhooks (with HMAC signing, retries, and rate limiting), or published to an MQTT topic for external consumers. Use this part to wire GoPLC into an observability stack, trigger downstream automation, or build your own operational dashboard.
